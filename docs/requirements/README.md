# Requirements

Active product/engineering requirements. A requirement records *what* is wanted
and *why*; active todos under `todos/` record *how* and progress. When a feature
ships, its completed requirement and durable todo knowledge move to
[`docs/features/`](../features/); Git history retains the work narrative.

Shipped-state docs include `thesis/README.md`,
`docs/MIGRATION-1.x-TO-2.x.md`, and `CHANGELOG.md`. Active decision and operating
records remain under `docs/`.

Conventions (owned by `.agents/skills/create-todo/SKILL.md`):

- File: `docs/requirements/<NN>-<requirement>.md` — next number = highest
  existing + 1.
- Creating a todo starts with reviewing this tree — update the matching
  requirement or create a new one, then cross-link both ways.

There are currently no active requirements.
